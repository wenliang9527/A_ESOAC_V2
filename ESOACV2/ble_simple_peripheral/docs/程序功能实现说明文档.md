## 一、项目概述

### 1.1 项目简介

ESOACV2是一款基于FR801xH平台的智能空调控制器固件，支持BLE、MQTT、UART三种通信方式，实现空调红外控制、状态监测、远程管理等功能。

### 1.2 主要功能特性

- **多协议通信**: BLE 5.0、MQTT over 4G、UART三通道统一协议
- **红外控制**: 学习/发送红外码，支持多种空调协议
- **状态监测**: 功率、温度实时采集，周期上报
- **设备管理**: 设备地址、BLE名称、MQTT配置可动态修改
- **OTA升级**: 支持远程固件升级
- **数据持久化**: 空调状态、红外按键、配置参数Flash存储

### 1.3 技术规格

| 项目 | 规格 |
|------|------|
| 主控芯片 | FR801xH (Cortex-M3, 48MHz) |
| 内存 | 128KB SRAM |
| Flash | 256KB (内部) + 外部SPI Flash |
| BLE版本 | 5.0 |
| 4G模组 | ML307A (MQTT over AT指令) |
| 红外载波 | 36kHz/38kHz/56kHz |
| ADC采样 | 10bit, 双通道 |

## 三、核心功能模块

### 3.1 协议处理模块 (protocol.c/h)

#### 3.1.1 功能概述

实现ESOAC通讯协议V2.2，统一处理BLE、MQTT、UART三通道的通信帧。

#### 3.1.2 V2.2协议帧格式

```
字节偏移   0      1      2           3          4-5           6-7           8 … (8+N-1)    最后 1 字节
          +------+------+-----------+----------+-------------+-------------+--------------+-----------+
          | 5A   | 7A   |data_length|data_mark | command(LE) | dev_addr(LE)|   Data (N)   | checksum  |
          +------+------+-----------+----------+-------------+-------------+--------------+-----------+
            帧头(固定)    L=5+N      请求/响应/   Cmd 小端      Addr 小端      业务数据      8 位累加和
                                           上报

整帧长度 = 4 + data_length = 9 + N (字节)
```

#### 3.1.3 关键技术点

**设备地址管理**:
```c
// 默认地址从芯片UID生成
uint16_t default_addr = (uint16_t)id_data.unique_id[4] | 
                        ((uint16_t)id_data.unique_id[5] << 8);

// Flash存储结构
typedef struct {
    uint32_t magic;      // 0x44455631 "DEV1"
    uint16_t dev_addr;
    uint16_t reserved;
} dev_addr_storage_t;
```

**地址过滤机制**:
```c
bool protocol_check_dev_addr(uint16_t dev_addr) {
    // 广播地址总是接受
    if (dev_addr == DEV_ADDR_BROADCAST) return true;
    // 检查本机地址
    return (dev_addr == dev_addr_get());
}
```

**校验和计算**:
```c
uint8_t protocol_calc_checksum(protocol_frame_t *frame) {
    uint8_t checksum = 0;
    // 帧头 + data_length + data_mark
    checksum += frame->frame_header[0];
    checksum += frame->frame_header[1];
    checksum += frame->data_length;
    checksum += frame->data_mark;
    // command (小端)
    checksum += (uint8_t)(frame->command & 0xFF);
    checksum += (uint8_t)((frame->command >> 8) & 0xFF);
    // dev_addr (V2.2新增)
    checksum += (uint8_t)(frame->dev_addr & 0xFF);
    checksum += (uint8_t)((frame->dev_addr >> 8) & 0xFF);
    // Data
    for (uint16_t i = 0; i < data_len; i++) {
        checksum += frame->data[i];
    }
    return checksum;
}
```

#### 3.1.4 命令处理流程

```
接收数据 → protocol_parse_frame() → protocol_process_frame()
                                              ↓
                                    地址过滤检查
                                              ↓
                                    switch(cmd)分发
                                              ↓
                                    ┌─────────┼─────────┐
                                    ↓         ↓         ↓
                                 空调控制   红外操作   配置管理
                                    ↓         ↓         ↓
                                 发送响应   发送响应   发送响应
```

#### 3.3.4 协议识别

```c
// 学习完成后自动识别协议类型
void ir_protocol_detect(TYPEDEFIRLEARNDATA *learn_data) {
    // 分析脉冲特征
    uint32_t *pulses = learn_data->ir_learn_Date;
    uint8_t count = learn_data->ir_learn_data_cnt;
    
    // NEC协议: 引导码 9ms低+4.5ms高
    if (is_nec_protocol(pulses, count)) {
        learn_data->protocol_type = IR_PROTO_NEC;
    }
    // GREE协议: 引导码 9ms低+4ms高
    else if (is_gree_protocol(pulses, count)) {
        learn_data->protocol_type = IR_PROTO_GREE;
    }
    // ... 其他协议识别
    
    // 识别按键功能
    learn_data->key_function = detect_key_function(pulses, count);
    
    // 温度键特殊处理
    if (learn_data->key_function == KEY_FUNC_TEMP_UP ||
        learn_data->key_function == KEY_FUNC_TEMP_DOWN) {
        learn_data->temp_ctrl_type = detect_temp_control_type(pulses, count);
        learn_data->temp_step = detect_temp_step(pulses, count);
    }
}
```

#### 3.3.5 发送控制

```c
void IR_start_send(TYPEDEFIRPWMTIM *IR_Send_struct) {
    // 1. 检查是否正在学习或发送
    if (ir_learn_data.IR_learn_state & 0x02) return;
    if (IR_PWM_TIM.IR_Busy) return;
    
    // 2. 配置PWM载波
    pwm_set_frequency(IR_Send_struct->ir_carrier_fre);
    pwm_start(PWM_CHANNEL_5);
    
    // 3. 启动Timer0微秒级定时
    timer0_init_us();
    timer0_start();
    
    // 4. 设置busy标志
    IR_PWM_TIM.IR_Busy = 1;
    IR_PWM_TIM.IR_pwm_Num = IR_Send_struct->ir_learn_data_cnt;
}

// Timer0中断服务程序
void timer0_isr_ram(void) {
    static uint8_t pulse_index = 0;
    
    if (IR_PWM_TIM.IR_pwm_state == 0) {
        // 载波开启
        pwm_set_duty(PWM_CHANNEL_5, 50);  // 50%占空比
        timer0_set_period(IR_PWM_TIM.IR_Pwm_State_Date[pulse_index++]);
        IR_PWM_TIM.IR_pwm_state = 1;
    } else {
        // 载波关闭
        pwm_set_duty(PWM_CHANNEL_5, 0);
        timer0_set_period(IR_PWM_TIM.IR_Pwm_State_Date[pulse_index++]);
        IR_PWM_TIM.IR_pwm_state = 0;
        
        // 检查是否发送完成
        if (pulse_index >= IR_PWM_TIM.IR_pwm_Num) {
            IR_send_complete();
        }
    }
}

    .ref_sel = ADC_REFERENCE_AVDD,  // AVDD参考

    .route.pad_to_sample = 1,        // PAD直通
    .clk_sel = ADC_SAMPLE_CLK_24M_DIV13,
    .clk_div = 0x3f,
};
```

#### 5.1.2 采样流程

```
启动ADC
  ↓
配置多通道轮询模式 (LOOP)
  ↓
等待转换完成
  ↓
读取双通道数据
  ↓
5点平均滤波
  ↓
物理量换算
  ├─→ 功率: P = (ADC * Vref / 1023) * Kp
  └─→ 温度: T = 1 / (1/T0 + ln(R/R0)/B) - 273.15
```

#### 5.1.3 滤波算法

```c
// 5点平均滤波
uint16_t adc_filter(uint16_t *samples, uint8_t count) {
    uint32_t sum = 0;
    for (int i = 0; i < count; i++) {
        sum += samples[i];
    }
    return (uint16_t)(sum / count);
}

// 采样流程
for (int i = 0; i < 5; i++) {
    adc_get_result(ADC_TRANS_SOURCE_PAD, 0x03, adc_buf);
    samples_power[i] = adc_buf[0];
    samples_ntc[i] = adc_buf[1];
    co_delay_10us(10000);  // 100ms间隔
}

ESAirdata.AIRpowerADCvalue = adc_filter(samples_power, 5);
ESAirdata.AIRntcADCvalue = adc_filter(samples_ntc, 5);
```

#### 5.2.1 存储映射

Flash地址空间 (W25Q128, 16MB)

#### 5.2.2 擦写保护

```c
// 扇区擦除 (4KB)
void SpiFlash_Erase_Sector(uint32_t sector_addr) {
    // 1. 写使能
    flash_write_enable();
    // 2. 发送扇区擦除命令
    flash_send_cmd(FLASH_SECTOR_ERASE, sector_addr);
    // 3. 等待完成
    flash_wait_busy();
}

// 页写入 (256B)
void SpiFlash_Write(uint8_t *data, uint32_t addr, uint32_t len) {
    while (len > 0) {
        uint32_t page_remain = 256 - (addr % 256);
        uint32_t write_len = (len < page_remain) ? len : page_remain;
        
        flash_write_enable();
        flash_page_program(data, addr, write_len);
        flash_wait_busy();
        
        addr += write_len;
        data += write_len;
        len -= write_len;
    }
}
```
