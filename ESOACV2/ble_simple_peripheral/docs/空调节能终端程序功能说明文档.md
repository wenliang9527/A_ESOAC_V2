# ESOAC 空调节能终端程序功能说明文档 (V2.2)

## 一、项目概述

### 1.1 产品简介

ESOAC（Energy Saving Of Air Conditioners）空调节能终端，基于 **FR801xH BLE SoC** 主控芯片和 **ML307A 4G 通信模组**，通过学习空调红外遥控信号实现空调的远程控制。设备通过 **BLE（蓝牙低功耗）** 和 **4G/MQTT** 双通道与上层应用通信，集成 NTC 温度传感器和电流互感器实现环境温度和功率的实时监测，为用户提供可视化的能耗数据和节能统计。

### 1.2 硬件平台

| 项目 | 参数 |
|------|------|
| 主控芯片 | FR8012H/FR8016H/FR8018H (富芮坤 BLE SoC, Cortex-M3, 48MHz) |
| 通信模组 | ML307A (4G，AT 指令驱动) |
| 温度传感器 | NTC 热敏电阻 (10kΩ @25°C)，ADC 采集 |
| 功率监测 | 电流互感器 + ADC 采集 |
| 红外收发 | PC5(发送/PWM5)、PC3(接收/EXTI_3) |
| 4G模组控制 | PA6(电源)、PA4(唤醒)、PA1(复位) |
| UART | UART0 → ML307A (115200bps)，UART1 → 调试 (115200bps) |
| SDK 版本 | FREQ BLE SDK V1.0.8 |
| 内存 | 128KB SRAM |
| Flash | 256KB (内部) + 外部SPI Flash (W25Q128, 16MB) |

### 1.3 应用场景

- 家庭空调节能管理
- 办公室空调集中控制
- 商业场所空调系统优化
- 智能家居集成控制

#### 3.1.1 V2.2协议帧格式

```
字节偏移:  0     1     2       3       4-5      6-7        8..(8+N-1)   最后1字节
           ┌─────┬─────┬───────┬───────┬────────┬──────────┬──────────┬───────┐
           │ 5A  │ 7A  │ Data  │ Data  │ Cmd    │ Dev_Addr │   Data   │ Check │
           │     │     │ Length│ Mark  │ (LE)   │ (LE)     │   (N)    │  sum  │
           └─────┴─────┴───────┴───────┴────────┴──────────┴──────────┴───────┘
                                           ↑         ↑           ↑
                                    命令字小端   设备地址小端   业务数据(最长250字节)

整帧长度 = 4 + data_length = 9 + N (字节)
```

**V2.2变更说明** (相对于V2.0):
- **新增字段**: `dev_addr` (2字节，小端序) - 设备短地址，支持广播地址 `0xFFFF`
- **data_length计算**: 从 `3 + N` 变为 `5 + N` (增加了dev_addr的2字节)
- **最大业务数据**: 从252字节减少到250字节
- **校验和范围**: 包含 `dev_addr` 字段

- **帧头**: `0x5A 0x7A` (2字节)
- **Data Length**: 从 Data Mark 到 Data 末尾的总字节数 = `5 + N`
- **Data Mark**: `0x00`=请求, `0x01`=响应, `0x02`=主动上报, `0xFF`=错误
- **Command**: 命令字，小端序 (Cmd_L 在前, Cmd_H 在后)
- **Dev Addr**: 设备短地址，小端序 (V2.2新增)
- **Data**: 业务数据 (最长 **250** 字节)
- **Checksum**: 从帧头到 Data 所有字节的 **8 位累加和**（包含dev_addr）

#### 3.1.3 MQTT配置 TLV 格式 (CMD_SET/GET_MQTT_CONFIG)

```
字节偏移: 0       1       2..N      N+1     N+2      ...
           ┌───────┬───────┬─────────┬───────┬─────────┐
           │ type  │ len   │ value   │ type  │  len    │ ...
           └───────┴───────┴─────────┴───────┴─────────┘
```

| type | 字段 | 最大长度 | 校验规则 |
|------|------|----------|----------|
| 0 | server_addr (服务器地址) | 127 | 可打印ASCII, 非空 |
| 1 | server_port (端口) | 5 | 可打印ASCII, 非空 |
| 2 | client_id (客户端ID) | 127 | 可打印ASCII, 非空 |
| 3 | username (用户名) | 127 | 可打印ASCII, 允许空(清空) |
| 4 | password (密码) | 127 | 宽松校验, 允许特殊字符 |
| 5 | subscribe_topic (订阅主题) | 127 | 可打印ASCII, 非空 |
| 6 | publish_topic (发布主题) | 127 | 可打印ASCII, 非空 |

> **特性**: 支持部分更新——仅提交需要修改的字段，未提交的字段保持原值不变。

#### 3.1.4 响应状态码

| 状态码 | 含义 |
|--------|------|
| `0x00` | 成功 (STATUS_SUCCESS) |
| `0x01` | 参数错误 (STATUS_ERROR_PARAM) |
| `0x02` | 未知命令 (STATUS_ERROR_CMD) |
| `0x03` | 帧累加和校验失败 (`STATUS_ERROR_CHECKSUM`，实现为 8 位和校验，非 CRC) |
| `0x04` | 设备忙 (STATUS_ERROR_BUSY) |
| `0x05` | 存储/Flash 错误 (STATUS_ERROR_STORAGE) |
| `0x06` | 设备地址错误 (STATUS_ERROR_ADDR) (V2.2新增) |
| `0xFF` | 执行失败 (STATUS_ERROR_FAIL) |

#### 3.1.5 设备地址管理 (V2.2新增)

**默认地址生成**:
```c
// 从芯片UID生成默认设备地址
uint16_t default_addr = (uint16_t)id_data.unique_id[4] | 
                        ((uint16_t)id_data.unique_id[5] << 8);
```

**广播地址**: `0xFFFF` - 所有设备都接受的地址

**地址过滤机制**:
```c
bool protocol_check_dev_addr(uint16_t dev_addr) {
    // 广播地址总是接受
    if (dev_addr == DEV_ADDR_BROADCAST) return true;
    // 检查是否为本机地址
    return (dev_addr == dev_addr_get());
}
```

**Flash存储结构**:
```c
typedef struct {
    uint32_t magic;      // 0x44455631 "DEV1"
    uint16_t dev_addr;   // 设备短地址
    uint16_t reserved;
} dev_addr_storage_t;
```

**地址错误处理**:
- 当请求帧中的 `dev_addr` 既非广播地址也非本机地址时，返回 `STATUS_ERROR_ADDR (0x06)`
- 响应帧始终使用本机地址作为 `dev_addr`

#### 3.1.6 响应帧格式 (CMD_RESPONSE 0x8000)

```
帧内data: [Cmd_H][Cmd_L][Status]
         2字节(原命令字大端序) + 1字节状态码
```

#### 3.1.8 多字节字段字节序（对接必读）

| 字段或场景 | 字节序 |
|------------|--------|
| 帧内 `command`（偏移 4–5） | **小端**（低字节在前） |
| 帧内 `dev_addr`（偏移 6–7） | **小端**（低字节在前） |
| `CMD_RESPONSE` (0x8000) 的 data 内嵌「原命令字」 | **大端**（高字节在前） |
| 心跳主动上报 4 字节计数器 | **大端** |
| `CMD_SYNC_TIME` 年份 16 位 | **大端** |

| `CMD_GET_POWER` / `CMD_GET_TEMP` | IEEE 754 float，**小端**（ARM 默认） |
| `CMD_SET_DEV_ADDR` / `CMD_GET_DEV_ADDR` 地址 | **小端** |

### 3.2 应用任务模块 (app_task.c / app_task.h)

#### 3.2.1 事件体系

| 事件ID | 名称 | 触发源 | 处理逻辑 |
|--------|------|--------|----------|
| `0x0101` | APP_EVT_MQTT_DATA_RECEIVED | MQTT消息到达 | 日志记录 |
| `0x0102` | APP_EVT_BLE_DATA_RECEIVED | BLE GATT写入 | 协议帧解析→命令分发 |
| `0x0103` | APP_EVT_MQTT_CONNECTED | MQTT连接成功 | 发送设备信息+状态到MQTT |
| `0x0104` | APP_EVT_MQTT_DISCONNECTED | MQTT断线 | 启动重连定时器 |
| `0x0105` | APP_EVT_SENSOR_DATA_READY | 传感器数据就绪 | (预留) |
| `0x0106` | APP_EVT_IR_LEARN_COMPLETED | 红外学习完成 | (预留) |
| `0x0107` | APP_EVT_IR_LEARN_FAILED | 红外学习失败 | (预留) |
| `0x0108` | APP_EVT_MQTT_CONFIG_UPDATED | MQTT配置已更新 | 断开旧连接→用新配置重连 |

#### 3.2.2 定时器体系

| 定时器 | 间隔 | 类型 | 功能 |
|--------|------|------|------|
| heartbeat_timer | 60s | 周期 | BLE心跳 + MQTT心跳(已连接时) |
| status_report_timer | 30s | 周期 | BLE状态上报 + MQTT状态上报(已连接时) |
| sensor_read_timer | 10s | 周期 | ADC采集→功率/温度计算→MQTT上报(已连接时) |
| mqtt_reconnect_timer | 30s/120s | 单次 | MQTT断线重连(递增策略) |

#### 3.2.3 MQTT断线重连策略

```
MQTT断线 (URC "+MQTTclosed:" 或 超时300s)
  │
  ▼
app_task_start_reconnect_timer()
  │  间隔: 30秒
  │
  ▼
mqtt_reconnect_timer_func()
  ├─ 重置状态机 ML307Con=NET_INITIALIZING
  ├─ 调用 ML307A_MQTTinit() 推进一步
  │
  ├─ 成功 → APP_EVT_MQTT_CONNECTED → stop_reconnect_timer()
  │
  └─ 失败 → reconnect_count++
       │
       └─ ≥10次 → 间隔延长至120秒, 计数重置
            │
            └─ 继续循环直到连接成功
```

#### 3.2.4 MQTT配置动态更新流程

```
BLE APP → CMD_SET_MQTT_CONFIG (TLV格式)
  │
  ▼
protocol_process_frame()
  ├─ 逐TLV解析 + validate_string_field() 校验
  ├─ 仅更新非空字段 → strncpy() 到 g_mqtt_config
  ├─ mqtt_config_save() → Flash 擦除→写入→回读验证
  ├─ protocol_send_response(SUCCESS)
  └─ app_task_send_event(APP_EVT_MQTT_CONFIG_UPDATED)
       │
       ▼
  app_task_func()
    ├─ stop_reconnect_timer()
    ├─ 若已连接: AT+MQTTDISC=0 → MLinitflag=ML307A_Idle
    ├─ ML307Con = DIS_CONNECTING (跳过网络注册, 复用网络)
    └─ start_reconnect_timer() (30s后用新配置重连)
```

#### 3.2.5 核心API接口

```c
// 初始化 (创建任务、加载配置、启动定时器)
```

#### 3.3.2 MQTT连接状态机


| 状态 | AT指令 | 说明 |
|------|--------|------|

### 3.8 空调状态持久化模块 (aircondata_persist.c / aircondata_persist.h)

#### 3.8.1 功能概述

#### 3.8.2 延迟保存机制

```c
// 状态修改后5秒自动保存，避免频繁擦写Flash
#define SAVE_DELAY_MS       5000    // 5秒延迟
#define SAVE_TIMER_PERIOD   100     // 100ms定时器周期

static bool save_pending = false;
static uint16_t save_delay_counter = 0;

void ESAirdata_TriggerSave(void) {
    save_pending = true;
    save_delay_counter = SAVE_DELAY_MS / SAVE_TIMER_PERIOD;  // 50
}

void ESAirdata_TimerHandler(void) {
    if (save_pending && --save_delay_counter == 0) {
        ESAirdata_Save();  // 执行实际保存
        save_pending = false;
    }
}
```

#### 3.8.3 数据校验机制

```c
typedef struct {
    uint32_t magic;          // 0x45534144 "ESAD"
    uint8_t version;         // 数据版本
    uint8_t reserved;
    uint16_t checksum;       // CRC16校验
    Airparameters data;      // 实际数据
} aircondata_persist_t;

// 校验流程
bool ESAirdata_Validate(const aircondata_persist_t *storage) {
    // 1. 魔数校验
    if (storage->magic != AIRCONDATA_MAGIC) return false;
    // 2. 版本校验
    if (storage->version != AIRCONDATA_VERSION) return false;
    // 3. CRC校验
    uint16_t calc_crc = crc16((uint8_t *)&storage->data, sizeof(Airparameters));
    if (storage->checksum != calc_crc) return false;
    // 4. 参数范围校验
    if (storage->data.AIRTemperature < 16 || storage->data.AIRTemperature > 30) return false;
    if (storage->data.AIRWindspeed > 2) return false;
    return true;
}
```

#### 3.8.4 上电加载流程

```

      ↓
ESAirdata_Load()
      ↓
从Flash读取数据
      ↓
ESAirdata_Validate() 校验

      └─→ 校验失败 → 使用默认值
                        ├─ 温度: 20°C
                        ├─ 模式: 制冷
                        ├─ 风速: 一级
                        └─ 开关: 关闭
```

### 3.9 红外发送后状态同步

#### 3.9.1 功能概述

红外发送完成后，根据发送的按键类型自动更新空调状态，保持状态一致性。

#### 3.9.2 状态更新逻辑

```c
void ESOAAIR_UpdateStatusByKey(uint8_t keynumber) {
    ir_key_function_t func = ESairkey.airbutton[keynumber].key_function;
    
    switch (func) {
        case KEY_FUNC_POWER:
            // 切换开关状态
            ESAirdata.AIRStatus = (ESAirdata.AIRStatus == airSW_ON) ? 
                                  airSW_OFF : airSW_ON;
            break;
            
        case KEY_FUNC_MODE:
            // 循环模式: 自动→制冷→制热→除湿→送风→睡眠→自动
            ESAirdata.AIRMODE = (ESAirdata.AIRMODE + 1) % 6;
            break;
            
        case KEY_FUNC_TEMP_UP:
            // 温度+1，限制16-30度
            if (ESAirdata.AIRTemperature < 30) {
                ESAirdata.AIRTemperature++;
            }
            break;
            
        case KEY_FUNC_TEMP_DOWN:
            // 温度-1，限制16-30度
            if (ESAirdata.AIRTemperature > 16) {
                ESAirdata.AIRTemperature--;
            }
            break;
            
        case KEY_FUNC_WIND:
            // 循环风速: 自动→一级→二级→三级→自动
            ESAirdata.AIRWindspeed = (ESAirdata.AIRWindspeed + 1) % 4;
            break;
            
        // ... 其他按键处理
    }
    
    // 触发延迟保存
    ESAirdata_TriggerSave();
    
    // 更新最近按键记录
    ESAirdata.AIPKP = keynumber;
}
```

```
    ├─ checksum: 2字节 (CRC16)
    └─ data: Airparameters结构
  扇区20-31 (addr=0x14000-0x1FFFF): 红外按键数据 (ESairkey, ~48KB)
    ├─ 最多支持 AIRkeyNumber 个按键
    └─ 每个按键包含脉冲数据和协议识别信息
```

#### 3.6.2 Flash保护机制

所有写操作采用三步保护：**擦除扇区 → 写入数据 → 回读验证**

```
写入流程:
  SpiFlash_Erase_Sector(sector_addr)     // 擦除
  SpiFlash_Write(data, addr, len)        // 写入
  SpiFlash_Read(verify, addr, len)       // 回读
  memcmp(verify, data, len) == 0 ?       // 校验
    → SUCCESS : FAIL
```

#### 3.6.3 数据加载校验

```
device_config_load():
  1. SpiFlash_Read() 读取Flash
  2. 校验 magic == 0x45534F43 (DEVICE_CONFIG_MAGIC)
     ├─ 不匹配 → 使用默认值 "ESOAC" → 写入Flash
     └─ 匹配 → 继续
  3. 二次校验: ble_name_len <= 20 且 ble_name[20] == '\0'
     ├─ 不通过 → 使用默认值 → 写入Flash
     └─ 通过 → 加载成功
```

```
mqtt_config_load():
  1. SpiFlash_Read() 读取Flash
  2. 校验 magic == 0x4D515454 (MQTT_CONFIG_MAGIC)
     ├─ 不匹配 → 保持 g_mqtt_config 默认值, return false
     └─ 匹配 → 继续
  3. 逐字段强制null终止 (防止溢出)
  4. strncpy() 拷贝到 g_mqtt_config → return true
```

#### 3.6.4 BLE名称校验规则

```c
ble_name_status_t device_config_validate_name(const char *name, uint8_t len);
```

| 校验项 | 规则 |
|--------|------|
| 空指针 | name == NULL → NAME_ERR_EMPTY |
| 空长度 | len == 0 → NAME_ERR_EMPTY |
| 超长 | len > 20 → NAME_ERR_TOO_LONG |
| 非法字符 | char < 0x20 或 > 0x7E → NAME_ERR_INVALID_CHAR |
| 全空格 | 所有字符均为空格 → NAME_ERR_EMPTY |
| 通过 | → NAME_OK |

## 四、数据流详解

### 4.1 BLE数据接收 → 命令处理

```
手机APP → BLE GATT Write
  → ESAIRble_service 回调
  → app_task_send_event(APP_EVT_BLE_DATA_RECEIVED)
  → app_task_func()
    → protocol_parse_frame()     ← 验证帧头5A7A + 校验和
    → protocol_process_frame(&frame, 0)  ← source=0 (BLE)
      ├─ CMD_SET_POWER:   ESAirdata.AIRStatus → ESOAAIR_IRsend(0)
      ├─ CMD_SET_MODE:    ESAirdata.AIRMODE = mode
      ├─ CMD_SET_TEMP:    ESAirdata.AIRTemperature = temp (16~30)
      ├─ CMD_SET_WIND:    ESAirdata.AIRWindspeed = speed
      ├─ CMD_SET_BLE_NAME: ble_update_device_name() → Flash → 重启广播
      ├─ CMD_SET_MQTT_CONFIG: TLV解析 → g_mqtt_config → Flash → 重连
      ├─ CMD_IR_LEARN_START: ESOAAIR_IRskeystudy(key)
      ├─ CMD_IR_SEND:     ESOAAIR_IRsend(key)
      └─ ...
    → protocol_send_response(CMD, STATUS, 0)  ← 回复BLE
```

### 4.2 MQTT下行数据 → 命令处理

```
云平台 → 4G → ML307A → UART0 RX (逐字节中断)
  → [5ms超时] uart0_recv_timer_func()
    → 检测 "+MQTTrecv:" → URC分流
      → 解析 linkid, topic, len
      → 定位 \r\n 后的数据体
      → mqtt_handler_message_arrived(topic, data, len)
        → protocol_parse_frame(data, len, &frame)
        → protocol_process_frame(&frame, 1)  ← source=1 (MQTT)
        → data_callback(data, len)           ← 用户回调
```

### 4.3 MQTT上行数据 (传感器上报)

```
sensor_read_timer (10s周期)
  → fr_ADC_send()                        ← ADC采集
  → Get_Power_Value(adc) → ESAirdata.AIRpowervalue
  → Get_Temperature_Value(adc) → ESAirdata.temp_celsius
  → if MLinitflag == ML307AMQTT_OK:
      mqtt_handler_send_power()    → protocol_send_power(1)
        → protocol_build_and_send(CMD_GET_POWER, DATA_MARK_RESPONSE, &float, 4, SOURCE_MQTT)
          → protocol_send_buffer()
            → ML307A_MQTTPublish(publish_topic, buffer, len)
              → AT+MQTTPUB=0,"topic",len → 等待">" → 逐字节发送 → 等待"OK"
      mqtt_handler_send_temperature()  ← 同上流程
```

### 4.4 双通道统一发送机制

```c
static void protocol_send_buffer(uint8_t *buffer, uint16_t len, uint8_t source)
{
    if (source == SOURCE_BLE) {
        ESAIR_gatt_report_notify(0, buffer, len);         // BLE GATT Notify
    } else if (source == SOURCE_MQTT) {
        if (R_atcommand.MLinitflag == ML307AMQTT_OK) {
            ML307A_MQTTPublish(g_mqtt_config.publish_topic, buffer, len);  // MQTT发布
        }
    }
    // SOURCE_UART: 调试通道暂不发送
}
```

## 六、用户操作流程

### 6.1 红外学习流程

```
1. APP发送 CMD_IR_LEARN_START (data[0]=按键索引)
   → 校验: data_length>=3, key<AIRkeyNumber, 未在学习中
   → ESOAAIR_IRskeystudy(key) 进入学习模式
   → 返回 STATUS_SUCCESS / STATUS_ERROR_BUSY

2. 用户对准设备按遥控器按键 (红外信号被捕获)

3. APP发送 CMD_IR_LEARN_STOP
   → IR_stop_learn() 停止学习

4. APP发送 CMD_IR_READ_DATA (data[0]=按键索引)
   → process_ir_data(key) 读取学习数据

5. APP发送 CMD_IR_SAVE_KEYS
   → ESOAAIR_Savekey() 保存到Flash

6. APP发送 CMD_IR_SEND (data[0]=按键索引) 测试
   → ESOAAIR_IRsend(key) 发送红外信号
```

### 6.2 空调远程控制流程

```
APP (BLE或MQTT) → CMD_SET_POWER(0x0101)  data[0]=1  → 空调开机
APP (BLE或MQTT) → CMD_SET_MODE(0x0102)   data[0]=1  → 制冷模式
APP (BLE或MQTT) → CMD_SET_TEMP(0x0103)   data[0]=26 → 26°C
APP (BLE或MQTT) → CMD_SET_WIND(0x0104)   data[0]=2  → 中风
APP (BLE或MQTT) → CMD_SET_POWER(0x0101)  data[0]=0  → 空调关机

→ 每条命令均返回 CMD_RESPONSE (0x8000) + 状态码
```

### 6.3 MQTT配置修改流程

```
1. APP发送 CMD_GET_MQTT_CONFIG (0x0504)
   ← 返回 TLV 格式的当前配置

2. APP构造 TLV 数据, 发送 CMD_SET_MQTT_CONFIG (0x0503)
   TLV: [type=0][len=20]["broker.example.com"]
        [type=1][len=4]["1883"]
        [type=6][len=15]["device/data/up"]
   → 校验每个字段
   → 更新 g_mqtt_config (仅更新非空字段)
   → mqtt_config_save() 持久化到Flash (擦除→写入→验证)
   ← 返回 STATUS_SUCCESS
   → 自动触发断开重连 (使用新配置)

3. 重连完成后, APP发送 CMD_GET_MQTT_CONFIG 验证
```

### 6.4 BLE名称修改流程

```
APP发送 CMD_SET_BLE_NAME (0x0501)
  data[0] = 名称长度
  data[1..N] = 名称字符串 (可打印ASCII, 最长20字节)

→ device_config_validate_name() 校验
→ device_config_set_ble_name() Flash保存
→ gap_stop_advertising()
→ gap_set_dev_name() + 更新scan_rsp_data
→ gap_start_advertising() 重启广播
← 返回 STATUS_SUCCESS / STATUS_ERROR_PARAM / STATUS_ERROR_FAIL
```

## 七、异常处理机制

### 7.1 协议层异常处理

| 异常场景 | 检测方式 | 处理策略 |
|----------|----------|----------|
| 帧长度不足 | `len < 9` (V2.2最小帧长) | `protocol_parse_frame()` 返回 false |
| 帧头不匹配 | buffer 中无 `0x5A 0x7A` | 返回 false |
| 校验和错误 | `protocol_calc_checksum() != checksum` | 打印日志, 返回 false |
| 数据超长 | `data_len > MAX_DATA_LENGTH` | `protocol_build_frame()` 返回 false |
| 未知命令 | switch default | 返回 STATUS_ERROR_CMD |
| 参数缺失 | `data_length < 5` (V2.2) | 返回 STATUS_ERROR_PARAM |
| 设备地址错误 | `!protocol_check_dev_addr(dev_addr)` | 返回 STATUS_ERROR_ADDR (V2.2新增) |
| 红外学习冲突 | `ESairkey.EIRlearnStatus == true` | 返回 STATUS_ERROR_BUSY |

### 7.2 MQTT连接异常处理

| 异常场景 | 检测方式 | 处理策略 |
|----------|----------|----------|
| URC断线通知 | UART0 收到 `+MQTTclosed:` | MLinitflag=Idle → 触发重连 |
| 首次连接超时 | MQTTinitout_timer (300s) | MLinitflag=ML307AMQTT_inttimeout |
| 重连超时 | reconnect_loop > 30000 | MLinitflag=ML307AMQTT_ERR |
| AT指令超时 | AT_WaitResponse() 返回1 | 状态机保持当前状态, 等待下次推进 |
| AT指令错误 | 收到 "ERROR" | 打印日志, 状态机不推进 |
| 发布失败 | ML307A_MQTTPublish() 返回false | 打印 "MQTT publish failed" |
| Flash保存失败 | 回读验证不通过 | 返回 STATUS_ERROR_FAIL |

### 7.3 Flash操作异常处理

| 异常场景 | 检测方式 | 处理策略 |
|----------|----------|----------|
| 魔数不匹配 | `magic != DEVICE_CONFIG_MAGIC` | 使用默认配置, 写入Flash |
| 数据损坏 | `ble_name_len > MAX` 或无null终止 | 使用默认配置, 写入Flash |
| 写入验证失败 | `memcmp(verify, data) != 0` | 返回false, 打印错误日志 |
| MQTT配置无数据 | `magic != MQTT_CONFIG_MAGIC` | 保持默认值, return false |
| 擦除验证失败 | 回读不全为0xFF | 返回false, 打印错误日志 |

### 7.4 配置更新异常处理

```
MQTT配置更新异常:
  Flash保存失败 → 返回 STATUS_ERROR_FAIL
                 → g_mqtt_config 已在RAM中更新但未持久化
                 → 不触发重连事件 (配置变更无效)
```
