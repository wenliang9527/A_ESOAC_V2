## 一、蓝牙功能介绍

### 1.1 功能概述

ESOACV2 蓝牙模块基于 BLE 5.0 技术，提供以下核心功能：

| 功能 | 说明 |
|------|------|
| **设备发现** | 广播设备名称，支持手机扫描连接 |
| **数据透传** | 支持手机与设备间的双向数据传输 |
| **状态查询** | 查询空调运行状态、功率、温度等 |
| **远程控制** | 开关机、模式切换、温度调节 |
| **OTA升级** | 支持空中固件升级 |
| **心跳上报** | 定时上报设备运行数据 |

### 1.2 技术参数

| 参数 | 值 |
|------|-----|
| 蓝牙版本 | BLE 5.0 |
| 工作模式 | 从机（Peripheral）模式 |
| 广播间隔 | 375ms |
| 发射功率 | 0dBm |
| 连接距离 | 约10米（视环境而定） |
| 单包数据 | 最大128字节 |
| 设备名称格式 | ESOAC + 12位地址（如 ESOAC330D3A1B2C3） |

### 1.3 服务与特征值

设备暴露以下GATT服务：

#### 标准服务

| 服务名称 | UUID | 说明 |
|----------|------|------|
| Generic Access | 0x1800 | 设备名称、外观、连接参数 |
| Generic Attribute | 0x1801 | 服务变更通知 |

#### 自定义服务

| 服务名称 | UUID | 说明 |
|----------|------|------|
| ESAIR主服务 | 0xFE10 | 数据通讯主服务 |
| OTA服务 | 0xFE00 | 固件升级服务 |

#### ESAIR主服务特征值

| 特征值 | UUID | 属性 | 功能 |
|--------|------|------|------|
| Version Info | 0xFF13 | Read | 读取协议版本 |
| Notify CCC | 0xFF12 | Read/Notify/Write | 通知使能控制 |
| TX Data | 0xFF10 | Read/Notify | 设备→手机数据 |
| RX Data | 0xFF11 | Write | 手机→设备数据 |

## 二、设备配对步骤

### 2.1 准备工作

**设备端**：
1. 确保 ESOACV2 设备已上电
2. 设备上电后自动开始蓝牙广播
3. 观察设备指示灯（如有），确认蓝牙模块正常工作

**手机端**：
1. 确保手机蓝牙已开启
2. 下载并安装 BLE调试工具（如 nRF Connect、LightBlue 等）
3. 确保手机在设备10米范围内

### 2.2 连接步骤

#### 步骤1：扫描设备

1. 打开 BLE调试APP
2. 点击 **"扫描"** 或 **"Scan"** 按钮
3. 在扫描结果列表中找到以 **"ESOAC"** 开头的设备名称
4. 设备名称格式示例：`ESOAC330D3A1B2C3`

> **提示**：如果找不到设备，请检查设备是否上电，或尝试靠近设备后重新扫描。

#### 步骤2：建立连接

1. 点击扫描到的设备名称
2. APP显示 **"正在连接..."**
3. 连接成功后，界面显示：
   - 设备名称（如 `ESOAC330D3...`）
   - 连接状态：**"蓝牙连接成功"**
   - **"断开连接"** 按钮

#### 步骤3：服务发现

连接成功后，APP自动执行服务发现，显示：
- **Generic Access** 服务（0x1800）
- **Generic Attribute** 服务（0x1801）
- **Unknown Service**（0xFE00 - OTA服务）
- **Unknown Service**（0xFE10 - ESAIR主服务）

> **说明**：自定义服务显示为 "Unknown Service" 是正常现象，因为它们是厂商自定义的SIG非标准服务。

### 2.3 连接验证

连接成功后，建议进行以下验证：

1. 展开 **Generic Access** 服务
2. 点击 **Device Name** 特征值的 **↓（读取）** 按钮
3. 确认读取到的名称与扫描时看到的名称一致
4. 验证连接正常

## 三、数据通讯操作

### 3.1 接收设备数据（必须先使能通知）

#### 步骤1：使能通知（CCC）

**重要**：在接收设备数据前，必须先使能通知功能。

1. 在 **Services** 标签页，展开 **UUID: 02f00000-0000-0000-0000-00000000fe10** 服务
2. 找到 **UUID: 02f00000-0000-0000-0000-00000000ff12** 的特征值
3. 点击展开该特征值
4. 在 **Descriptors** 下找到 **"Client Characteristic Configuration"**
5. 点击右侧的 **↑（写入）** 图标
6. 选择 **HEX格式**
7. 输入值：`0100`（表示使能Notify）
8. 点击确认写入

**验证使能成功**：
- 写入后，设备端日志会显示：`ESAIR CCC notify EN`
- 此时设备可以向手机发送数据

#### 步骤2：查看接收数据

1. 切换到 **Log** 标签页
2. 等待设备发送数据（如心跳包）
3. 在Log中查看接收到的HEX数据

**示例心跳包数据**：
```
5A 7A 11 01 01 00 34 12 XX XX XX XX XX XX XX XX XX XX XX XX
│  │  │  │  │  │  │  │  └─ 12字节业务数据
│  │  │  │  │  │  │  └──── 设备地址高字节
│  │  │  │  │  │  └─────── 设备地址低字节
│  │  │  │  │  └────────── 命令（0x0001=心跳）
│  │  │  │  └───────────── 数据标记（0x01=响应）
│  │  │  └──────────────── 数据长度（0x11=17）
│  │  └─────────────────── 帧头2
│  └────────────────────── 帧头1
└───────────────────────── 帧头1
```

### 3.2 发送命令到设备

#### 步骤1：找到RX特征值

1. 在 **Services** 标签页，展开 **UUID: 02f00000-0000-0000-0000-00000000fe10** 服务
2. 找到 **UUID: 02f00000-0000-0000-0000-00000000ff11** 的特征值
3. 确认属性包含 **WRITE** 或 **WRITE WRITE NO RESPONSE**

#### 步骤2：构造协议帧

ESOACV2 使用 V2.2 协议格式：

```
| 字段 | 长度 | 值 | 说明 |
|------|------|-----|------|
| 帧头1 | 1B | 0x5A | 固定帧头 |
| 帧头2 | 1B | 0x7A | 固定帧头 |
| 数据长度 | 1B | 5+N | 5 + 业务数据长度 |
| 数据标记 | 1B | 0x00 | 0x00=请求, 0x01=响应, 0x02=通知 |
| 命令 | 2B | - | 小端格式 |
| 设备地址 | 2B | - | 小端格式（可用0xFFFF广播） |
| 业务数据 | N B | - | 可选 |
| 校验和 | 1B | - | 前面所有字节的累加和 |
```

#### 步骤3：常用命令示例

**命令1：心跳请求（CMD_HEARTBEAT = 0x0001）**

假设设备地址为 `0x1234`：

```
帧内容：5A 7A 05 00 01 00 34 12
计算校验和：
  5A + 7A + 05 + 00 + 01 + 00 + 34 + 12 = 0x0120
  取低字节：0x20

完整帧：5A 7A 05 00 01 00 34 12 20
```

**HEX输入**：`5A7A050100341220`

---

**命令2：查询状态（CMD_GET_STATUS = 0x0201）**

```
帧内容：5A 7A 05 00 01 02 34 12
校验和：5A + 7A + 05 + 00 + 01 + 02 + 34 + 12 = 0x0122 → 0x22

完整帧：5A 7A 05 00 01 02 34 12 22
```

**HEX输入**：`5A7A050102341222`

---

**命令3：查询设备信息（CMD_DEVICE_INFO = 0x0002）**

```
帧内容：5A 7A 05 00 02 00 34 12
校验和：5A + 7A + 05 + 00 + 02 + 00 + 34 + 12 = 0x0121 → 0x21

完整帧：5A 7A 05 00 02 00 34 12 21
```

**HEX输入**：`5A7A050200341221`

---

**命令4：开机（CMD_SET_POWER = 0x0101，数据=0x01）**

```
帧内容：5A 7A 06 00 01 01 34 12 01
校验和：5A + 7A + 06 + 00 + 01 + 01 + 34 + 12 + 01 = 0x0134 → 0x34

完整帧：5A 7A 06 00 01 01 34 12 01 34
```

**HEX输入**：`5A7A0600010134120134`

---

**命令5：关机（CMD_SET_POWER = 0x0101，数据=0x00）**

```
帧内容：5A 7A 06 00 01 01 34 12 00
校验和：5A + 7A + 06 + 00 + 01 + 01 + 34 + 12 + 00 = 0x0133 → 0x33

完整帧：5A 7A 06 00 01 01 34 12 00 33
```

**HEX输入**：`5A7A0600010134120033`

#### 步骤4：执行写入

1. 点击 **0xFF11** 特征值右侧的 **↑（写入）** 图标
2. 选择 **HEX格式**
3. 输入完整的协议帧（如 `5A7A050100341220`）
4. 选择写入类型：
   - **Write**（带响应，推荐）- 等待设备返回确认
   - **Write Without Response**（无响应）- 更快但不确认
5. 点击确认发送

#### 步骤5：查看响应

1. 切换到 **Log** 标签页
2. 查看设备返回的数据
3. 根据命令类型解析响应数据

## 四、界面元素说明

### 4.1 顶部状态栏

| 元素 | 含义 | 说明 |
|------|------|------|
| 设备名称 | `ESOAC330D3...` | 连接的设备名称，可能因长度被截断 |
| 连接状态 | `蓝牙连接成功` | 绿色背景表示连接正常 |
| 断开连接按钮 | `断开连接` | 点击断开当前连接 |

### 4.2 Services标签页

| 元素 | 含义 | 操作 |
|------|------|------|
| ▶ 或 ▼ | 展开/折叠箭头 | 点击展开查看服务内的特征值 |
| Service名称 | 服务名称 | 标准服务显示名称，自定义服务显示"Unknown Service" |
| UUID | 服务唯一标识 | 128位UUID，用于识别服务类型 |
| PRIMARY SERVICE | 主服务标记 | 表示这是一个主服务（非包含服务） |

### 4.3 特征值列表

| 元素 | 含义 | 说明 |
|------|------|------|
| Characteristic名称 | 特征值名称 | 标准特征值显示名称，自定义显示"Unknown Characteristic" |
| UUID | 特征值唯一标识 | 128位UUID |
| Properties | 属性 | 支持的操作类型 |
| ↓ | 可读 | 点击读取当前值 |
| ↑ | 可写 | 点击写入新值 |
| ↓↑ | 可读可写 | 支持读写两种操作 |

### 4.4 属性类型说明

| 属性 | 全称 | 功能 |
|------|------|------|
| READ | 读 | 可以读取特征值当前值 |
| WRITE | 写 | 可以写入数据（带响应） |
| WRITE NO RESPONSE | 无响应写 | 写入数据但不等待响应 |
| NOTIFY | 通知 | 设备可以主动推送数据 |
| INDICATE | 指示 | 类似Notify，但需确认接收 |

### 4.5 描述符（Descriptors）

| 描述符 | UUID | 功能 |
|--------|------|------|
| Client Characteristic Configuration | 0x2902 | 使能/禁用Notify或Indicate |
| Characteristic User Description | 0x2901 | 特征值的用户描述文字 |

## 五、常见问题排查

### 5.1 扫描不到设备

| 可能原因 | 排查方法 | 解决方案 |
|----------|----------|----------|
| 设备未上电 | 检查设备电源指示灯 | 确保设备正常上电 |
| 距离过远 | 靠近设备（<1米）再扫描 | 移动到设备附近 |
| 广播间隔问题 | 等待几秒后重新扫描 | 多次尝试扫描 |
| 手机蓝牙异常 | 关闭再开启手机蓝牙 | 重启手机蓝牙 |
| APP权限不足 | 检查APP定位权限 | 授予定位权限（Android） |
| 设备已连接 | 检查是否有其他手机已连接 | 断开其他连接后重试 |

### 5.2 连接失败

| 可能原因 | 排查方法 | 解决方案 |
|----------|----------|----------|
| 距离过远 | 靠近设备 | 移动到1米范围内 |
| 信号干扰 | 远离WiFi路由器、微波炉 | 更换环境位置 |
| 设备连接数满 | 检查是否已有设备连接 | 断开其他连接 |
| 配对信息冲突 | 删除手机蓝牙配对记录 | 在系统蓝牙设置中删除配对 |
| 设备异常 | 重启设备 | 断电重启设备 |

### 5.3 无法接收数据

| 可能原因 | 排查方法 | 解决方案 |
|----------|----------|----------|
| 未使能CCC | 检查是否写入0x0100到CCC | 按步骤使能通知 |
| CCC写入错误 | 确认写入值格式 | 使用HEX格式写入0100 |
| 写入到错误的CCC | 确认是0xFF12特征的CCC | 展开正确的特征值 |
| 设备未发送数据 | 检查设备端日志 | 确认设备有数据上报 |
| 连接断开 | 检查连接状态 | 重新连接设备 |

### 5.4 发送命令无响应

| 可能原因 | 排查方法 | 解决方案 |
|----------|----------|----------|
| 写入到错误的特征值 | 确认是0xFF11（RX） | 选择正确的特征值 |
| 协议帧格式错误 | 检查帧头、校验和 | 按协议格式重新构造 |
| 设备地址不匹配 | 确认使用正确的设备地址 | 使用0xFFFF广播地址测试 |
| 数据格式错误 | 确认使用HEX格式 | 切换为HEX格式输入 |
| 命令不支持 | 检查命令码是否正确 | 参考命令列表使用正确命令 |

### 5.5 连接不稳定/频繁断开

| 可能原因 | 排查方法 | 解决方案 |
|----------|----------|----------|
| 距离过远 | 观察信号强度 | 保持设备在3米范围内 |
| 电池电量低 | 检查设备电源 | 更换电池或连接电源 |
| 环境干扰 | 观察周围电子设备 | 远离干扰源 |
| 连接参数不适配 | 检查连接间隔 | 等待设备自动更新连接参数 |
| 手机系统问题 | 尝试其他手机 | 更换手机测试 |

## 六、故障解决方案

### 6.1 完全无法连接

**现象**：扫描不到设备，或连接立即失败

**解决步骤**：

1. **检查设备状态**
   - 确认设备已正常上电
   - 观察设备指示灯（如有）
   - 确认设备未处于升级模式

2. **检查手机设置**
   - 确认手机蓝牙已开启
   - 授予APP定位权限（Android必需）
   - 关闭手机省电模式

3. **清除配对记录**
   - 进入手机系统设置 → 蓝牙
   - 找到历史配对记录中的ESOAC设备
   - 删除配对记录
   - 重新扫描连接

4. **重启设备**
   - 断开设备电源
   - 等待5秒后重新上电
   - 等待设备启动完成（约3秒）
   - 重新扫描连接

5. **更换APP测试**
   - 尝试使用其他BLE调试APP
   - 推荐：nRF Connect、LightBlue、BLE Scanner

### 6.2 能连接但无法通讯

**现象**：连接成功，但收不到数据或发送无响应

**解决步骤**：

1. **验证CCC使能**
   - 确认已向0xFF12特征的CCC写入0x0100
   - 重新使能一次
   - 检查Log中是否有数据接收

2. **检查协议格式**
   - 确认帧头为 `5A 7A`
   - 确认校验和计算正确
   - 使用广播地址 `0xFFFF` 测试

3. **简化测试**
   - 先发送简单的查询命令（如心跳请求）
   - 确认能收到响应后再发送控制命令

4. **查看设备日志**
   - 通过串口连接设备
   - 查看设备端接收和发送日志
   - 确认设备收到命令并处理了

### 6.3 数据接收异常

**现象**：能收到数据，但数据内容不正确

**解决步骤**：

1. **检查数据格式**
   - 确认使用HEX格式查看
   - 不要误用ASCII格式

2. **校验数据完整性**
   - 检查帧头是否为 `5A 7A`
   - 检查数据长度字段
   - 验证校验和

3. **解析响应数据**
   - 参考协议文档解析各字段
   - 注意小端字节序

### 6.4 特定命令失败

**现象**：某些命令执行失败，返回错误码

**错误码对照表**：

| 错误码 | 含义 | 解决方案 |
|--------|------|----------|
| 0x01 | 参数错误 | 检查命令参数是否在有效范围内 |
| 0x02 | 命令不支持 | 使用正确的命令码 |
| 0x03 | 校验和错误 | 重新计算校验和 |
| 0x04 | 设备忙 | 等待当前操作完成后再发送 |
| 0x05 | 存储失败 | 检查Flash是否正常 |
| 0x06 | 地址不匹配 | 使用正确的设备地址 |
| 0xFF | 通用失败 | 检查设备状态和命令格式 |

## 七、高级使用技巧

### 7.1 使用广播地址测试

当不确定设备地址时，可以使用广播地址 `0xFFFF`：

```
帧示例：5A 7A 05 00 01 00 FF FF XX
                              │  │
                              │  └── 地址高字节（0xFF）
                              └───── 地址低字节（0xFF）
```

**注意**：广播地址会被所有设备接收，多设备环境下慎用。

### 7.2 批量命令发送

如需连续发送多个命令，建议：
1. 每条命令间隔 100ms 以上
2. 等待上一条命令响应后再发送下一条
3. 避免同时发送多条控制命令

### 7.3 数据解析技巧

**心跳包数据解析示例**：

```
接收数据：5A 7A 11 01 01 00 34 12 00 00 80 41 00 00 A0 41 01 01 1A 00 8C

解析：
- 帧头：5A 7A ✓
- 数据长度：0x11 = 17（5 + 12字节数据）✓
- 数据标记：0x01 = 响应 ✓
- 命令：0x0001 = 心跳 ✓
- 设备地址：0x1234 ✓
- 业务数据（12字节）：
  - 功率（float，4字节）：00 00 80 41 = 16.0W
  - 温度（float，4字节）：00 00 A0 41 = 20.0°C
  - 电源状态（1字节）：01 = 开机
  - 模式（1字节）：01 = 制冷
  - 设定温度（1字节）：1A = 26°C
  - 风速（1字节）：00 = 低风
- 校验和：8C（需验证）
```

### 7.4 日志记录与分析

建议开启APP的日志记录功能：

### B. 校验和计算工具（Python示例）

```python
def calc_checksum(data):
    """计算ESOACV2协议校验和"""
    return sum(data) & 0xFF

# 示例：构造心跳请求帧
def make_frame(cmd, dev_addr, data=[]):
    header = [0x5A, 0x7A]
    length = 5 + len(data)
    mark = 0x00  # 请求
    cmd_lo = cmd & 0xFF
    cmd_hi = (cmd >> 8) & 0xFF
    addr_lo = dev_addr & 0xFF
    addr_hi = (dev_addr >> 8) & 0xFF
    
    frame = header + [length, mark, cmd_lo, cmd_hi, addr_lo, addr_hi] + data
    checksum = calc_checksum(frame)
    frame.append(checksum)
    
    return bytes(frame)

# 生成心跳请求帧（设备地址0x1234）
frame = make_frame(0x0001, 0x1234)
print(frame.hex())  # 输出: 5a7a050100341220
```
