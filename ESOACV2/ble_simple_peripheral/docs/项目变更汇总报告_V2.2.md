## 一、变更概述

## 二、协议V2.2核心变更点

| 变更项 | V1版本 | V2.2版本 | 影响 |
|--------|--------|----------|------|
| data_length | = 3 + N | = 5 + N | 帧结构变化 |
| dev_addr字段 | 无 | 2字节，小端 | 新增设备寻址 |
| 最大业务数据 | 252字节 | 250字节 | 容量减少 |
| 校验和范围 | 不含dev_addr | 含dev_addr | 校验计算变化 |
| 广播地址 | 无定义 | 0xFFFF | 新增广播功能 |
| 地址错误响应 | 无 | STATUS_ERROR_ADDR(0x06) | 新增错误码 |
| 设置/获取短地址命令 | 无 | 0x0505/0x0506 | 新增命令 |

#### 1. ESOACSum函数更新 (第18-33行)
```c
/* 协议帧校验和计算（V2.2版本）
 * 计算范围：帧头(2B) + data_length(1B) + data_mark(1B) + command(2B) + dev_addr(2B) + Data(N)
 */
uint8_t ESOACSum(TCommDataPacket buf)
{
    uint8_t ByteSum = 0;
    uint16_t i;

    /* 校验和计算范围：从帧头(偏移0)到校验字节前(偏移FramLen-1) */
    for (i = 0; i < buf.FramLen - 1; i++) {
        ByteSum += buf.FrameBuf[i];
    }

    return ByteSum;
}
```

## 十一、URC处理优化方案实施记录（2026-03-28）

### 11.1 变更背景

**问题描述**:
- 原有URC处理机制存在代码重复问题（`+MQTTclosed`处理逻辑在多处重复）
- URC解析与AT响应处理混杂在UART接收函数中，代码臃肿
- 新增URC类型需要修改多处代码，维护成本高
- 原有设计的状态机框架未启用（`ML307A_ProcessURC`依赖的`urc_ready`标志无代码置位）

**优化目标**:
1. 统一URC处理入口，消除代码重复
2. 支持异步处理，避免阻塞UART接收
3. 易于扩展，新增URC类型只需添加前缀和处理器

### 11.2 具体修改内容

#### 11.2.1 新增文件

| 文件路径 | 功能说明 | 代码行数 |
|---------|---------|---------|
| `usercode/urc_parser.h` | URC解析器头文件，定义URC类型、队列结构、函数声明 | 97行 |
| `usercode/urc_parser.c` | URC解析器实现，包含队列操作、URC解析、分发处理 | 425行 |

**核心数据结构**:

**关键代码变更**:

1. **UART接收处理简化** (`frusart.c`):
```c
/* 优化前: 85行，直接处理URC */
void uart0_recv_timer_func(void *arg) {
    // ... 直接解析+MQTTrecv和+MQTTclosed ...
}

/* 优化后: 35行，URC入队异步处理 */
void uart0_recv_timer_func(void *arg) {
    urc_entry_t entry;
    if (urc_parse(ATUSART_0_RXbuf.FrameBuf, rx_len, &entry)) {
        urc_queue_push(&R_atcommand.urc_queue, &entry);
    } else {
        R_atcommand.RECompleted = true;
    }
}
```

2. **URC定时器驱动** (`frATcode.c`):
```c
/* URC处理定时器回调 - 50ms周期 */
static void urc_timer_handler(void *arg) {
    urc_entry_t entry;
    uint8_t processed_count = 0;
    while (urc_queue_pop(&R_atcommand.urc_queue, &entry) && processed_count < 4) {
        urc_process_entry(&entry);
        processed_count++;
    }
}
```

3. **统一URC分发** (`urc_parser.c`):
```c
void urc_process_entry(const urc_entry_t *entry) {
    switch (entry->type) {
        case URC_MQTT_RECV:
            mqtt_handler_message_arrived(entry->topic, (uint8_t *)entry->data, entry->data_len);
            break;
        case URC_MQTT_CLOSED:
            // 统一处理，消除重复代码
            if (R_atcommand.MLinitflag == ML307AMQTT_OK) {
                R_atcommand.MLinitflag = ML307A_Idle;
                mqtt_handler_connection_callback(false);
                app_task_send_event(APP_EVT_MQTT_DISCONNECTED, NULL, 0);
            }
            break;
        // ... 其他URC类型处理 ...
    }
}
```
