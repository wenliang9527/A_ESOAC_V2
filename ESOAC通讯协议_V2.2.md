第 1 章  协议帧与校验和（V2 版本）
================================================================================

1.1  data_length 定义（V2）
--------------------------------------------------------------------------------

data_length 表示从 Data Mark 起，到最后 1 个业务数据字节为止的字节数，包含：
  data_mark(1) + command(2) + dev_addr(2) + Data(N)

故：
  data_length = 5 + N（其中 N 为业务载荷长度，N ≥ 0，N 最大 250）

字段为 uint8，须 5+N ≤ 255，故 N 最大 250。

1.2  线格式与偏移（V2）
--------------------------------------------------------------------------------

字节偏移   0      1      2           3          4-5           6-7           8 … (8+N-1)    最后 1 字节
          +------+------+-----------+----------+-------------+-------------+--------------+-----------+
          | 5A   | 7A   |data_length|data_mark | command(LE) | dev_addr(LE)|   Data (N)   | checksum  |
          +------+------+-----------+----------+-------------+-------------+--------------+-----------+
            帧头(固定)    L=5+N      请求/响应/   Cmd 小端      Addr 小端      业务数据      8 位累加和
                                           上报

整帧长度（字节）= 4 + data_length

1.3  校验和计算（checksum）
--------------------------------------------------------------------------------

计算范围（8 位无符号累加和，非 CRC）：
  帧头 2 字节(0x5A, 0x7A)
  + data_length
  + data_mark
  + command 低字节 + command 高字节
  + dev_addr 低字节 + dev_addr 高字节
  + data 全部字节

1.4  数据标记 data_mark
--------------------------------------------------------------------------------

取值     名称         说明
------   -----------  --------------------------------------------------------
0x00     请求         上位机 → 设备
0x01     响应         设备 → 上位机（应答某命令）
0x02     主动上报     设备定时/事件推送
0xFF     错误         错误类标记（与响应状态码配合使用）

第 2 章  命令字与载荷
================================================================================

命令字为 16 位无符号数；线路上以低字节在前（小端）。下表按功能域分组。

2.1  基础与心跳
--------------------------------------------------------------------------------

命令码     名称             方向     数据格式 / 说明
--------   ---------------  -------  ----------------------------------------
0x0001     心跳             双向     主动上报：4B 大端递增计数器
0x0002     设备信息         查询     应答：6B 设备ID + 4B 软件版本 + 4B 硬件版本

2.2  空调控制
--------------------------------------------------------------------------------

命令码     名称             方向     数据格式 / 说明
--------   ---------------  -------  ----------------------------------------
0x0101     设置开关         设置     data[0]: 0关 1开
0x0102     设置模式         设置     data[0]: 0自动 1制冷 2制热 3除湿 4送风 5睡眠
0x0103     设置温度         设置     data[0]: 16~30 ℃
0x0104     设置风速         设置     data[0]: 风速等级（枚举）

2.3  状态与传感器
--------------------------------------------------------------------------------

命令码     名称             方向     数据格式 / 说明
--------   ---------------  -------  ----------------------------------------
0x0201     获取状态         查询     应答 5B：开关/模式/温度/风速/连接
0x0202     获取功率         查询     应答 4B float 小端 IEEE754
0x0203     获取温度         查询     应答 4B float 小端 IEEE754
0x0204     时间同步         设置     8B：年16位大端+月日时分秒周各1B
0x0205     获取ADC          查询     应答 4B：两路 uint16 小端原始值

2.4  红外
--------------------------------------------------------------------------------

命令码     名称             方向     数据格式 / 说明
--------   ---------------  -------  ----------------------------------------
0x0301     红外学习开始     设置     data[0]: 按键索引
0x0302     红外学习停止     设置     无载荷
0x0303     红外发送         设置     data[0]: 按键索引
0x0304     红外学习结果     —        预留；固件未单独处理
0x0305     读取红外数据     查询     data[0]: 按键索引
0x0306     保存红外按键     设置     无载荷

2.5  网络与设备名
--------------------------------------------------------------------------------

命令码     名称             方向     数据格式 / 说明
--------   ---------------  -------  ----------------------------------------
0x0401     设置定时器       —        预留，未实现
0x0402     获取定时器       —        预留，未实现
0x0501     设置BLE名称      设置     data[0]=长度, 后跟名称字节
0x0502     获取BLE名称      查询     应答同上格式
0x0503     设置MQTT配置     设置     TLV，见第 3 章
0x0504     获取MQTT配置     查询     应答 TLV

2.6  设备短地址（V2 新增）
--------------------------------------------------------------------------------

命令码     名称             方向     数据格式 / 说明
--------   ---------------  -------  ----------------------------------------
0x0505     设置设备短地址   设置     data[0]=Addr_L, data[1]=Addr_H（小端）
0x0506     获取设备短地址   查询     应答 2B：Addr_L, Addr_H

2.7  OTA 与通用应答
--------------------------------------------------------------------------------

命令码     名称             方向     数据格式 / 说明
--------   ---------------  -------  ----------------------------------------
0x0601     OTA开始          —        预留，未实现
0x0602     OTA数据          —        预留，未实现
0x0603     OTA结束          —        预留，未实现

================================================================================
第 3 章  MQTT 配置 TLV（0x0503 / 0x0504）
================================================================================

载荷由若干 TLV 顺序拼接组成，可只携带需要修改的字段（部分更新）。

格式：
  [ type(1B) ][ len(1B) ][ value × len ] [ type ][ len ][ value ] …

示例：仅改服务器与端口 → 两个 TLV 即可，其余字段保持设备内原值。

type     字段             最大 len     校验说明
------   ---------------  -----------  ----------------------------------------
0        server_addr      127          可打印 ASCII，非空
1        server_port      5            可打印 ASCII，非空，如 1883
2        client_id        127          可打印 ASCII，非空
3        username         127          可打印 ASCII，允许空
4        password         127          允许特殊字符
5        subscribe_topic  127          可打印 ASCII，非空
6        publish_topic    127          可打印 ASCII，非空

注意：蓝牙名称不属于 MQTT TLV，请使用命令 0x0501 / 0x0502。

================================================================================
第 4 章  响应状态码与多字节序
================================================================================

4.1  通用应答中的状态字节
--------------------------------------------------------------------------------

值       符号（代码）                含义
------   -------------------------  ----------------------------------------
0x00     STATUS_SUCCESS             成功
0x01     STATUS_ERROR_PARAM         参数错误
0x02     STATUS_ERROR_CMD           未知命令
0x03     STATUS_ERROR_CHECKSUM      帧累加和错误（非 CRC）
0x04     STATUS_ERROR_BUSY          设备忙
0x05     STATUS_ERROR_STORAGE       存储/Flash 错误
0x06     STATUS_ERROR_ADDR          设备短地址不匹配（V2 新增）
0xFF     STATUS_ERROR_FAIL          执行失败

4.2  多字节字段字节序
--------------------------------------------------------------------------------

场景                                 字节序     备注
-----------------------------------  ---------  --------------------------------
帧内 command（偏移 4–5）              小端       与 ARM 自然序一致
帧内 dev_addr（偏移 6–7）             小端       与 command 一致
dev_addr 默认值（取自 UID 最后 2B）    小端       Addr_L=UID[4], Addr_H=UID[5]
CMD_RESPONSE 内嵌原命令字             大端       data[0]=高字节
心跳 NOTIFY 计数器（4B）              大端       与帧内 command 约定不同
CMD_SYNC_TIME 年份 16 位              大端       data[0] 为高字节
float、ADC uint16                     小端       IEEE754 / 原始采样值

================================================================================
第 5 章  设备短地址（dev_addr）详细说明
================================================================================

5.1  目的与适用范围
--------------------------------------------------------------------------------

为在 UART、BLE、MQTT 等同一二进制帧格式下，标识报文对应的空调节能终端
实例，在保持帧头 0x5A 0x7A 不变的前提下，在固定字段区增加 2 字节设备短
地址 dev_addr。

5.2  与既有标识的关系
--------------------------------------------------------------------------------

? 芯片 efuse 得到的 6 字节唯一 ID 仍作为全局设备标识（与现有设备信息命令
  中的 6B ID 一致）。
? dev_addr 为 16 位短地址，用于同一总线/同一项目内的快速寻址与报文标注；
  不保证全球唯一。
? 短地址默认值取 6 字节唯一 ID 的最后 2 字节，线序为小端。

5.3  线格式中的位置
--------------------------------------------------------------------------------

data_mark（1B）→ command（2B，小端）→ dev_addr（2B，小端）→ Data（N B）

整帧结构：
  0x5A 0x7A | data_length | data_mark | Cmd_L Cmd_H | Addr_L Addr_H | Data[N] | checksum

5.4  取值范围与默认值
--------------------------------------------------------------------------------

项目           约定
-------------  ------------------------------------------------------------
取值范围       0x0000～0xFFFF；其中 0xFFFF 保留为广播地址
默认值         上电后未配置时：dev_addr = UID[4] | (UID[5] << 8)
               其中 UID[0..5] 为 efuse 读出的 6 字节芯片唯一 ID
修改方式       通过 0x0505/0x0506 命令经 UART/BLE/MQTT 下发/查询
持久化         SET 成功后写入 SPI Flash，下次上电以存储值为准

5.5  接收与过滤规则
--------------------------------------------------------------------------------

5.5.1 请求帧（data_mark = 0x00）

? 若 dev_addr 等于本机当前配置的短地址：正常解析并执行命令。
? 若 dev_addr = 0xFFFF（广播）：本机接受该帧并执行（用于一对多场景）。
? 若 dev_addr 既不等于本机地址，也不等于广播保留值：不得执行命令，
  返回 CMD_RESPONSE（0x8000）状态码 STATUS_ERROR_ADDR（0x06）。

5.5.2 响应帧（data_mark = 0x01）与主动上报（data_mark = 0x02）

设备发出的响应与上报帧中 dev_addr 必须为本机当前短地址，以便主机在多
设备环境下区分来源。

第 6 章  传输通道与处理流程
================================================================================

同一解析入口保证 BLE、MQTT、UART 行为一致，便于测试与维护。

                        +------------------+
                        |   外部接口        |
                        +--------+---------+
                                 |
        +------------------------+------------------------+
        |                        |                        |
   +----+-----+            +-----+----+            +------+-------+
   | BLE GATT |            |  MQTT    |            | UART1        |
   +----+-----+            +-----+----+            +------+-------+
        |                        |                        |
        +------------------------+------------------------+
                                 |
                                 v  原始字节流（V2 帧格式）
                    +------------------------+
                    | protocol_parse_frame() |
                    +-----------+------------+
                                |
                                v
                    +------------------------+
                    | protocol_verify_checksum|
                    +-----------+------------+
                                |
                                v
                    +------------------------+
                    |  dev_addr 过滤检查      |
                    |  (不匹配则返回错误)     |
                    +-----------+------------+
                                |
                                v
                    +------------------------+
                    | protocol_process_frame()|
                    |  --> 命令分发 / 应答    |
                    +------------------------+

固件源码路径：ESOACV2/ble_simple_peripheral/usercode/protocol.c、protocol.h
